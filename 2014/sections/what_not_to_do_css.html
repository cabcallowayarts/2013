What Not to Do

  • Inline CSS

    If you look more in-depth in CSS, there are three main ways that you can use it: inline, in-document, and embedded. As you probably saw in the title, the one of these that you want to avoid is inline. Of course, you need to know what you're looking for before you know to avoid it, so here's an example:

    <p style="color:red;font-size:16px;width:100%;">Hello World</p>

    The way this works is that an element gets a new attribute, the style attribute, and it's value is set to CSS, with each style being separated by semicolons. This seems harmless at first, but when applied to a large-scale project, it becomes a horrible, un-expandable mess of code that no one will want to touch. The two acceptable ways of embedding CSS into a webpage are in the styles tags, like this, in the head tag:

    <style> p{
    color: red;
    } </style>

    Or in a separate file, and then embedded using the link tag in the head tag, like this:

    <link rel="stylesheet" href="index.css">
  • Using the measurements pc, in, cm, or any real-world measurement

    This one may seem a bit weird at first, but this rule becomes very, very important once you start designing for the real world. These measurements, pc (picas), in(inches), and cm(centimeters), were developed in the 20th century, when most screens were pretty much the same size, and you could get away with giving exact measurements. Nowadays, with the rise of laptops, phones, and tablets, screen sizes can vary from 3 inches to 27 inches, and you can't use the same measurement for both. Currently, the same measurements, which I have used so far, are px (pixels) and % (percent), though even pixels can become a bit risky if you are designing large elements. There are some other measurements out there, such as em, which are acceptable, but if you can measure it with a ruler, it is best to not use it.
